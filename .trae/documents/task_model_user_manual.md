# 任务模型（新版）用户操作手册

## 1. 核心概念

- 项目（PROJECT）：顶层对象，带指派部门与创建部门信息；可指定 1 名项目负责人。
- 操作任务（OPERATION_TASK）：属于某个项目；支持分配多个操作员并行执行；可拆分同类型子任务。
- 质检：不再创建独立质检任务（QA_TASK 已移除），改为源任务状态流转——操作员提交质检后，质检员接收并判定通过或不通过。

所有任务支持可选的计划完成时间（plannedDueAt）。输入需为标准 ISO-8601 时间字符串（例如 `2026-02-15T12:00:00Z`），为空则表示未计划。

## 2. 角色与边界

- 生产管理员（PRODUCTION_MANAGER）
  - 全局项目统筹；可跨部门分配项目给部门管理员。
- 部门管理员（DEPARTMENT_ADMIN）
  - 可接收生产管理员分配的项目；
  - 可创建本部门新项目；
  - 可指定/变更项目负责人（项目经理）。
- 项目经理（PROJECT_MANAGER）
  - 只能被部门管理员指派项目，禁止创建项目；
  - 可在被指派的项目内创建操作任务及任务序列。
- 操作员（OPERATOR）
  - 仅允许接收分配的操作任务；
  - 仅允许在已分配的任务下创建同类型子任务；
  - 可提交质检和撤销质检。
- 质检员（INSPECTOR）
  - 在待质检泳道接收质检任务；
  - 在质检中泳道判定通过或不通过。

## 3. 项目创建与负责人指定

1. 进入管理看板，点击"新建项目"。
2. 填写项目名称、描述。
3. 在"项目负责人"中选择 1 名负责人（可为空，后续再指派）。
4. 提交后即生成一个"项目"顶层任务，系统记录项目创建人、创建部门，并以指派部门作为部门隔离依据。

## 4. 操作任务创建与多人指派

1. 进入项目详情，点击"分发新任务"。
2. 填写任务名称。
3. 在"指派给（作业员，可多选）"中选择多个操作员。
4. 提交后生成操作任务。

## 5. 操作任务拆分与质检流程

- 操作员在执行过程中，如果需要拆分任务，可在已分配的操作任务下创建同类型子任务。
- 操作员在"进行中"状态累计输入完成量；完成量达到总工作量时，"输入完成量"按钮变为"提交质检"按钮。
- 点击"提交质检"后，任务进入"待质检"泳道。

## 6. 质检流程

质检流程分为两步：接收 → 判定。

### 6.1 操作员视角

| 状态 | 可见按钮 | 操作说明 |
|------|----------|----------|
| 进行中（完成量=总工作量） | 提交质检 | 点击后任务进入"待质检"泳道 |
| 待质检 | 撤销 | 点击后任务回退到"进行中"，工作量退回，保留已填完成量 |

> 操作员仅可在"待质检"状态撤销质检；质检员接收后（质检中）不可撤销。

### 6.2 质检员视角

| 状态 | 可见按钮 | 操作说明 |
|------|----------|----------|
| 待质检 | 接收 | 点击后任务进入"质检中"泳道，assigneeId 转为质检员 |
| 质检中 | 通过 | 点击后任务进入"质检完成"泳道 |
| 质检中 | 不通过 | 点击后任务回退到"进行中"，工作量退回，保留已填完成量 |

### 6.3 质检推送

- 未指定质检员时，任务推送给质检部门负责人（具有 `quality:check` 权限的部门中具有 `department:manager` 权限的用户）。
- 已指定质检员时，任务推送给该质检员，出现在质检员的待质检泳道。

## 7. 状态流转总览

```
待处理 → 待接收 → 已接收 → 进行中 → 待质检 → 质检中 → 质检完成 → 完成
                    ↑                  │        │
                    └── 撤销(操作员)    │        └── 不通过(质检员) → 进行中
                    └──────────────────┘
```
